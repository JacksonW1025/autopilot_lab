# 阶段工作总结与下一步研究计划

**研究方向**：基于参数敏感度与飞行包线保护的无人机（UAV）安全飞控算法研究

## 1. 课题背景与研究动机

本课题建立在 ISSTA 2025 杰出论文 RouthSearch (px4_ws/reference)的研究基础之上。RouthSearch 结合了劳斯-赫尔维茨（Routh-Hurwitz）稳定性判据与坐标搜索算法，解决了由于用户在起飞前“错误配置”三维 PID 参数（比例、积分、微分）而导致的无人机失控坠毁问题，为飞控程序推断出了静态的连续有效参数边界规范 。



然而，在实际飞行中，即使 PID 参数配置完全正确且处于静态安全边界内，**人类驾驶员或上层自动规划算法给出的“极端/激进控制指令（Aggressive Maneuvers）”**依然可能导致无人机由于物理执行器受限而失控。因此，本课题的研究目标从“静态参数规范”向“动态控制指令的安全兜底”推进，旨在软件和算法层面设计一种飞行包线保护（Flight Envelope Protection）机制，确保在任何激进输入下无人机的绝对安全。

## 2. 理论基础：离散时间状态空间模型

本研究的核心理论抓手是经典的状态空间方程在底层的物理体现：

$$y_{t+1} = a x_t + b y_t$$

- **$y_t$ 与 $y_{t+1}$（状态变量）**：代表无人机在当前和下一时刻的实际物理姿态响应（如横滚角、俯仰角、角速度）。
- **$x_t$（输入变量）**：代表驾驶员或 ROS2 节点发送的控制指令（如瞬间要求 60 度翻滚的激进输入）。
- **$a$（控制输入影响矩阵）与 $b$（状态转移矩阵）**：代表飞控系统内的“参数向量”。

**核心研究问题（参数敏感度分析）：**

在常规平稳飞行中，$a$ 向量（包含比例增益、前馈控制）的作用是线性的。但在 $x_t$ 发生剧烈、高频的激进变化时，由于电机转速等物理硬件存在上限，某些飞控底层参数会爆发出极高的“敏感度”。例如，电机输出指令（ActuatorMotors）会瞬间达到 100% 发生**执行器饱和（Actuator Saturation）**。一旦饱和，$a$ 向量的有效值会断崖式下降，导致激进指令 $x_t$ 无法转化为期望的 $y_{t+1}$，进而引发严重的积分抗饱和（Integrator Windup）与驾驶员诱发震荡（PIO），这就是导致无人机在激进操作下失控的物理与算法根源。

## 4. 下一步工作与实操计划

为了准确找到“哪些参数对激进操作更敏感”，并最终提出创新算法，接下来的工作将分为四个阶段进行：

### 阶段一：构建并注入激进激励信号 (Aggressive Input Generation)

- **目标**：在 Gazebo 仿真中可复现地触发无人机的不稳定状态。
- **操作**：编写专用的 ROS2 测试节点，向 `fmu/in/vehicle_attitude_setpoint` 或 `fmu/in/manual_control_setpoint` 话题注入极端的控制信号。
- **信号设计**：采用阶跃信号（Step Input）或大范围的扫频信号（Sweep Input），例如瞬间给定要求横滚角达到物理极值，并保持极短时间后反拉。

### 阶段二：底层敏感参数的监控与捕获 (Data Logging & Monitoring)

- **目标**：观测激进输入 $x_t$ 下，$y_{t+1}$ 与底层控制约束的非线性畸变。
- **在线观测**：利用 `PlotJuggler` 工具实时订阅并绘制 ROS2 话题曲线，重点对比目标姿态与实际姿态的延迟（Phase Lag）。
- **离线高频分析**：提取 PX4 内部高频（如 250Hz以上）生成的 ULog 日志，并上传至 PX4 Flight Review 分析平台。
- **重点关注的敏感变量**：
  - `ActuatorMotors`（电机输出PWM/转速，寻找削顶饱和点）。
  - 姿态控制器的 Error 项积累与角加速度限幅触发情况。

### 阶段三：参数敏感度分析与失效点定位 (Sensitivity Analysis)

- **目标**：利用数据证明 $y_{t+1} = a x_t + b y_t$ 理论。
- **操作**：通过日志数据，绘制在不同变化率的 $x_t$ 下，电机输出饱和度与轨迹跟踪误差（Tracking Error）的发散曲线。明确界定出系统的“线性控制区”与“非线性失效区（危险包线边界）”。
